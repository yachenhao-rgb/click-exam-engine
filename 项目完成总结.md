# 题目质量验证功能 - 项目总结

## 📋 项目概述

为智能出题引擎实现了题目质量验证功能，提供自动化的题目质量评估，确保生成的完形填空题目符合中考英语标准。

## ✅ 已完成的功能

### 1. 核心验证模块
- **文件**: `core/question_validator.py`
- **类**: `QuestionValidator`
- **功能**: 
  - 单题验证
  - 批量验证
  - 验证报告生成

### 2. 验证规则 (5项)

#### ✅ 规则1: 词性标注验证
- 验证单词在语境中的词性是否正确
- 支持 NLTK 词性标注（可选）
- 使用词尾规则进行辅助判断
- 支持6种词性: noun, verb, adj, prep, conj, adv

#### ✅ 规则2: 干扰项词性一致性
- 检查干扰项是否为真实单词
- 检查干扰项词性是否与答案一致
- **创新点**: 包含同根词映射（如 success/successful 被视为可接受）
- 检测重复干扰项
- 检测与答案相同的干扰项

#### ✅ 规则3: 答案唯一性验证
- 检查干扰项是否在上下文中出现
- 确保答案是语境中的唯一合适选择
- 避免歧义

#### ✅ 规则4: 挖空位置合理性
- 检测是否挖空了专有名词
- 检测是否挖空了数字
- 检测是否挖空了单字母词
- 检测常见功能词（a, the, in等）

#### ✅ 规则5: 难度匹配验证
- 基于 CEFR 词表评估词汇难度
- 支持三个难度等级: easy, medium, hard
- 提供难度警告

### 3. 质量评分算法

**总分**: 0-10分

**评分维度与权重**:

| 维度 | 权重 | 说明 |
|------|------|------|
| 词性准确性 | 30% | 单词词性是否正确 |
| 干扰项质量 | 25% | 干扰项是否为真实单词且词性一致 |
| 答案唯一性 | 25% | 答案是否在语境中唯一 |
| 挖空合理性 | 10% | 挖空位置是否合理 |
| 难度匹配度 | 10% | 难度是否匹配目标年级 |

## 🎯 使用说明

### 基本使用

```python
from core.question_validator import QuestionValidator

# 创建验证器
validator = QuestionValidator()

# 验证单题
result = validator.validate_question({
    'correct_word': 'success',
    'options': ['success', 'succeed', 'successful', 'successfully'],
    'pos': 'noun',
    'context': 'Hard work leads to success.',
    'position': 4
})

print(f"评分: {result['score']}/10")
print(f"有效: {result['is_valid']}")
print(f"问题: {result['issues']}")
```

### 批量验证

```python
# 批量验证
batch_result = validator.validate_batch(questions)

# 生成报告
report = validator.generate_validation_report(batch_result)
print(report)
```

### 与出题引擎集成使用

```python
from core.exam_generator import ExamGenerator

generator = ExamGenerator()
result = generator.generate_exam(text, num_blanks=10)

# 自动包含质量评分
print(f"整体质量评分: {result['overall_score']}/10")

for q in result['questions']:
    print(f"第{q['number']}题: {q['quality_score']}/10")
```

## 🎓 验证规则详解

### 词性标注验证规则

1. **无上下文时**: 使用内置词库匹配
2. **有 NLTK 时**: 使用 NLTK 进行词性标注
3. **无 NLTK 时**: 使用词尾规则猜测
4. **比对结果**: 验证声称的词性与实际是否一致

### 干扰项一致性规则

1. **真实单词检查**: 基于 CEFR 词表和词尾规则
2. **同根词接受**: 常见同根词（如 success/successful）被视为可接受
3. **重复检查**: 检测完全相同的干扰项
4. **答案相同检查**: 检测与正确答案完全相同的干扰项

### 挖空位置规则

1. **专有名词检测**: 常见人名、地名等
2. **数字检测**: 所有数字
3. **单字母词检测**: 常见单字母词
4. **功能词警告**: 常见功能词（如 a, the）

## ❓ 常见问题

### Q: NLTK 未安装会怎样？
A: 验证器会使用简化的词尾规则进行验证，虽然准确度略有下降，但仍能提供可靠的验证结果。

### Q: 如何提高验证准确度？
A: 
1. 安装 NLTK
2. 定期更新 CEFR 词表
3. 根据实际使用情况调整评分权重

### Q: 验证结果不理想怎么办？
A: 
1. 检查具体的 `issues` 和 `warnings`
2. 根据提示调整题目设置
3. 参考改进建议进行优化

### Q: 如何忽略某些验证规则？
A: 可以修改 `QuestionValidator` 类的相应验证方法，或者在调用时过滤结果。
